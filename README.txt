Desafio criado para exercitar o display flex com o uso do posicionamento com flexbox.

Vocabulário:
    > Pragma: dadi, fato, pré-requisito para o argumento.
    > Alent: foster, nourish, feed, maintain, develop.
    > HiDPI (High Dots Per Inch): são telas com alta resolução em um formato relativamente pequeno. São geralmente encontrados em high-end laptops e monitores.

Definição de elementos, atributos e valores
    > http-equiv (atr.): define uma diretiva pragma (alent).
    > X-UA-Compatible (val.): escolhe a versão do Internet Exploter no qual a página deve ser renderizada.
    > viewport (val.): valor para 'name' que dá dicas sobre o tamanho do tamanho inicial da janela de exibição. Usado apenas por dispositivos móveis.
    > width (val.): valor para 'content', subvalor de 'viewport. Um número inteiro, ou o texto 'device-width'. Define a lagura de pixels da janela de exibição em que você quer que o site seja renderizado.
    > initial-scale (val.): valor para 'content', subvalor de 'viewport'. Um número positivo entre 0.0 e 10.0. Define a razão entre a largura do dispositivo ('device-width' no modo retrato ou 'device-height' no modo paisagem) e o tamanho da janela de exibição.
    > viewport (em dispositivo): é o espaço que o dispositivo emula ter, não a resolução real em pixels que tem a tela.

Conceitos

pixels: um pixel de conteúdo é renderizado com mais de um pixel físico. se você desenhar uma linha de 1px na tela (com CSS ou mesmo uma app nativa), ela será renderizada como uma linha de 2px físicos. Isso melhora, a definição dos elementos na tela. Existe, então, uma razão de multiplicação entre pixels lógicos e pixels físicos chamada de device pixel ratio. E tudo isso se conclui no que realmente importa, no tamanho da viewport. Que basicamente é o quanto de conteúdo cabe na tela, em CSS pixels (pixel lógico), não device pixels (pixel fisíco).
